Overview

The Shuri API is a powerful and easy-to-use application programming interface that provides highly secure global data services to relationship management apps. It provides fast, secure and global data storage web scale, and is based on a simple Relationship Management pattern. In addition, the API's unique multiple database capability makes collaborating and sharing data easy.

Relationship Management Model


API Resources

The API has been built from the ground up to support the data storage needs of any relationship management application by providing a simpler, more concise, yet still incredibly useful vocabulary - all based entirely on the model.

Resources exposed by The API includes 3 "core" entities and several "attribute" entities:
  • People
  • Touches
  • Groups
  • Attributes
    • Tags
    • ContactPoints
    • Locations
    • Documents
One additional entity, UserTypes, enables extensibility and customization for developers. With just these 8 entities, the API supports a very wide range of relationship management functionality.


Key API Concepts:
  1. Every instance of any entity must be associated to one database.
  2. Instances are owned by a single person. Optionally, instances may be also owned by a single team. Only owners may update the instance.
  3. Any database may be "published" to a Team whose members may then subscribe. Subscribers get "read-only" privileges to everything in the database.
  4. No practical limit to the number of instances of any entity in The API.
  5. Instances are identified by a globally unique id.
