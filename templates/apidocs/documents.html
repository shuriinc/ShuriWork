Documents

The entity named Document* is used within the API in the very broad sense of: all attributes other than Contact Point, Location, or Tag.

Documents, along with the robust set of pre-defined document UserTypes provide flexibility and extensibility to The API. The Document field value stores up to 4000 characters, of which the first 900 are indexed.

Documents are used to store custom fields, files, notes, ratings, and other data types which are defined by the primitive per the document's usertype. Documents can be related to a person, organization, touch, a person on a particular touch, and/or belong to a private group. Every document has one usertype and that usertype will be of one of the available document primitive types, listed below.


Resource Description
GET document?id=[ id ] Returns a fully hydrated document.
POST documents
?subscriptionId=[ group id ]
Create a new document in the collection requested or the user's private collection by default. If the id is empty a create occurs, else an update to the existing record.
DELETE document Deletes an existing document. The document and all of it's relations are permanently deleted. User must own the document or be a member of the owning group.
* The entity name Document was chosen instead of Attribute because Attribute is a key word in many technologies.

Document Primitives

All document usertypes must be defined as one of the primitives. Documents have a name (140 characters max) and a value (4000 characters max.) What each contains and how it is displayed and handled is determined by the primitive.

Primitives make it easy for developers extend the API to capture any information or media required in their solution.
Primitive Name Description and Usage
Note Simple name-value pair
File Name is the human-friendly name; Value contains a Json File structure //TBD
Text Field holding text and expecting a single-line for input and display; Value holds the text; Name can hold the label or be blank (the usertype name is the label)
LongText Field holding text and expecting multiple lines for input and display; Value holds the text; Name can hold the label or be blank (the usertype name is the label)
Integer Field for numeric integers. Value holds the number; Name can hold the label or be blank (the usertype name is the label)
Float Field for non-integer numbers. Value holds the number; Name can hold the label or be blank (the usertype name is the label)
Date Field for non-integer numbers. Value holds the date; Name can hold the label or be blank (the usertype name is the label)
RatingYesNo Field for binary rating like Thumb Up/Down. Value holds either 0 or 1; Name can hold the label or be blank (the usertype name is the label)
RatingYesNoMaybe Field for three-mode rating. Value holds either 0, 1, or 2; Name can hold the label or be blank (the usertype name is the label)
Rating0to5 Field for scale of 1-5 rating, i.e. stars. Or 0, which is 'not rated'. Value holds integer 0 to 5; Name can hold the label or be blank (the usertype name is the label)
Rating0to100 Field for scale of 1-100 rating. Or 0, which is 'not rated'. Value holds integer 0 to 100; Name can hold the label or be blank (the usertype name is the label)
CurrencyUS Field for money in US dollars. Value holds the dollar amount; Name can hold the label or be blank (the usertype name is the label)
Object Value holds a JSON object
