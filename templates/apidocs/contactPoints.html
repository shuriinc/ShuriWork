ContactPoints

ContactPoints are things like email addresses, phone numbers, urls, etc. ContactPoints are associated to people and groups. The name field of a contactPoint contains its actual value. How it is displayed and handled is determined by the primitive. Every contactPoint has one usertype and that usertype will be of one of the available contactPoint primitive types, listed below.

Resource Description
GET contactpoints
?entityType=[ entityType ]
&entityId=[ id ]
Returns an array of all contact points that are related to the entity requested. Paginated.
GET contactpoint?id=[ id ] Returns a fully hydrated contact point.
POST contactpoints
?subscriptionId=[ group id ]
Create a new contact point in the collection requested or the user's private collection by default. If the id is empty a create occurs, else an update to the existing record.
DELETE contactpoint Deletes an existing contact point. The contact point and all of it's relations are permanently deleted. User must own the contact point or be a member of the owning group.

ContactPoint Primitives

All contactPoint usertypes must be defined as one of the primitives.

Primitive Name Description and Usage
PrimaryEmail Main email address for a person or organization
Email Any other email address
Phone Any phone number - should be complete + international-dial-compliant
Url Any web url associated to any person or organization
SMHandle Any social media handle, such as a Twitter screenname, Facebook, LinkedIn. The API provides public UserTypes for the most common social media. See below.
